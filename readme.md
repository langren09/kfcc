### 使用git进行版本管理
- 创建 dev 分支作为开发分支
- 注意管理好分支，一般情况不要修改`master`，避免与 dev 分支造成冲突
- 每个分支的的日志是不同的
- 分支没有修改后内容后没有提交是无法操作切换分支的
- 版本回退： `git reset --hard 版本号前几位`
